Atelier elasticsearch.

1. Installer elasticsearch

Vous pouvez télécharger eleasticsearch 0.20.6 ici. Attention ! La version 0.90.0.RC1 n’est pas compatible avec tous les plugins. Nommer votre cluster avec votre nom dans le fichier elastisearch.yml (param cluster.name) pour éviter les conflits avec vos petits camarades !

Solution 
$ wget https://download.elasticsearch.org/elasticsearch/elasticsearch/elasticsearch-0.20.6.tar.gz
$ tar -xvf elasticsearch-0.20.6.tar.gz
$ cd elasticsearch-0.20.6/config/
$ vi elasticsearch.yml

4. Afficher le contenu de l’index

Pour s'assurer que les documents sont bien dans l'index, afficher le contenu de l'index foo.

Hint
Vous pouvez trouver la documentation de l'api search ici
Solution 
$curl -XGET 'http://localhost:9200/foo/_search' -d '{
  "query" : {
    "match_all" : {}
  }
}'
ou encore plus simple 
$curl -XGET 'http://localhost:9200/foo/_search'

5. Chercher des documents (1/3)

Faire une recherche pour trouver le document qui contient dans le champ text america en faisant une requête dans l'url et ensuite une requête "match" en JSON

Hint
Vous pouvez trouver la documentation de la query DSL ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?q=text:america'
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "match" : {
      "text" : "america"
    }
  }
}'

6. Chercher des documents (2/3)

Faire une recherche avec comme critère ameri*a en faisant une requête wildcard et ensuite avec comme critère ameriqa en faisant une recherche fuzzy

Hint
Vous pouvez trouver la documentation des queries fuzzy ici et wildcard ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "wildcard" : {
      "text" : "ameri*a"
    }
  }
}'
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "fuzzy" : {
      "text" : "ameriqa"
    }
  }
}'

7. Chercher des documents (3/3)

Faire une recherche sur l'ensemble de l'index en faisant une facet qui donne le count des 20 termes les plus fréquents dans le champ text

Hint
Vous pouvez trouver la documentation de la facet terms ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "match_all" : { }
  }, 
  "facets" : {
    "countTerms" : {
      "terms" : {
        "field" : "text",
        "size": 20
      }
    }
  }
}'

8. Altérer la réponse

Faire la recherche pour trouver le document qui contient america tout en formattant la réponse pour qu'elle soit lisible humainement, et afficher l'explication du scoring.

Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?pretty=true' -d '{
  "explain": true,
  "query" : {
    "match" : {
      "text" : "america"
    }
  }
}'

11. Initialiser la river mongodb (1/3)

Avant d'initialiser la river mongo, pour permettre l'interprétation du champ location du document mongo, initialiser l'index bank, avec des documents de type customers et customiser le mapping du champ location pour qu'il soit de type geo_point

Hint
Avant de pouvoir définir le mapping, il faut créer l'index. Vous pouvez trouver de la documentation pour le mapping geo_point ici
Solution 
$ curl -XPUT 'http://localhost:9200/bank'
$ curl -XPUT 'http://localhost:9200/bank/customers/_mapping' -d '{
    "customers" : {
      "properties":{
        "location": { "type" : "geo_point", "lat_lon" : true }
      }
    }
}'

12. Initialiser la river mongodb (3/3)

Initialiser la river mongodb pour créer l'index bank, avec des documents de type customers

Hint
Vous pouvez trouver de la documentation ici
Solution 
$ curl -XPOST 'http://localhost:9200/_river/bank_river/_meta' -d '{
  "type": "mongodb",
    "mongodb": {
      "db": "test-es", 
      "collection": "customers"
    },
    "index": {
      "name": "bank", 
      "type": "customers"
    }
}'
L'activation de la river mongodb se manifest avec des logs dans la console. On peut aussi voir l'activité sur le plugin bigdesk. L'indexation doit prendre moins de 5 minutes.

14. Chercher des documents (avancé 1/2)

Ecrire une requête qui retourne les personnes dont leur enrollmentDate est avant le 1er janvier 2000 (strictement) triés du plus récent au plus ancien en comptant le nombre de compte encore ouvert (traduit par le champ closingDate dans account à null).

Hint
Dans une facet terms, le champ missing donne le nombre de documents où le champ indiqué dans la facet n'est pas rempli.
Solution 
$ curl -XGET 'http://localhost:9200/bank/customers/_search?pretty=true' -d '{
  "query" : {
    "range" : { 
      "enrollmentDate" : { "lt" : "2000-01-01" }
    }
  }, 
  "sort" : { "enrollmentDate" : "desc" },
  "facets" : {
    "repartitionCompte" : {
      "terms" : { "field" : "closingDate" }
    }
  }
}'

15. Chercher des documents(avancé 2/2)

Chercher les personnes qui se trouvent à moins de 20km de Paris.

Hint
Les coordonnées de Paris sont "48.8742, 2.3470". Il faut appliquer un filter geo_distance sur le résultat de la query, la doc ici
Solution 
$ curl -XGET 'http://localhost:9200/bank/customers/_search?pretty=true' -d '{
  "query": {
    "match_all": {}
  },
  "filter": {
    "geo_distance": {
      "distance": "20km",
      "location": "48.8742, 2.3470"
    }
  }
}'
