Atelier elasticsearch.

1. Installer elasticsearch

Vous pouvez télécharger eleasticsearch 1.1.0 ici. Nommer votre cluster avec votre nom dans le fichier elastisearch.yml (param cluster.name) pour éviter les conflits avec vos petits camarades !

Solution 
$ curl -O https://download.elasticsearch.org/elasticsearch/elasticsearch/elasticsearch-1.1.0.tar.gz
$ tar -xvf elasticsearch-1.1.0.tar.gz
$ cd elasticsearch-1.1.0/config/
$ vi elasticsearch.yml
(repérer la ligne #cluster.name: elasticsearch, la décommenter et mettre un nom de cluster unique à vous.)

2. Lancer elasticsearch

Vous pouvez dès à présent lancer elasticsearch via la ligne de commande. Par défaut vous pouvez suivre les logs dans la console.

Solution 
$ cd ../bin/
$ ./elasticsearch
On voit dans les logs que le port d'écoute par défaut d'elasticsearch est le 9200. On peut s'assurer qu'elasticsearch est bien démarré faisant un curl http://localhost:9200/ (ou via votre browser). Vous devez y voir un JSON recapitulant les caractéristiques de votre serveur.

3. (OPTIONNEL) Installer le plugin marvel (si vous voulez éviter de faire du curl)

Marvel a un dashboard nommé Sense qui permet de faire des requête sur votre elasticsearch directement depuis votre browser. Cela permet d'avoir une vraie colorisation et validation syntaxique de vos requêtes. Cerise sur le gâteau, le copier coller de la syntaxe curl fonctionne ! Ne pas oublier de redémarrer elasticsearch après l'installation !

Solution 
$ ./plugin -install elasticsearch/marvel/latest

5. Afficher le contenu de l’index

Pour s'assurer que les documents sont bien dans l'index, afficher le contenu de l'index foo en utilisant l'api search.

Hint
Vous pouvez trouver la documentation de l'api search ici
Solution 
$ curl -XGET 'http://localhost:9200/foo/_search' -d '{
  "query" : {
    "match_all" : {}
  }
}'
ou encore plus simple 
$ curl -XGET 'http://localhost:9200/foo/_search'

6. Chercher des documents (1/3)

Faire une recherche pour trouver le document qui contient dans le champ text america en faisant une requête dans l'url et ensuite une requête "match" en JSON

Hint
Vous pouvez trouver la documentation de la query DSL ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?q=text:america'
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "match" : {
      "text" : "america"
    }
  }
}'

7. Chercher des documents (2/3)

Faire une recherche avec comme critère ameri*a en faisant une requête wildcard et ensuite avec comme critère ameriqa en faisant une recherche fuzzy

Hint
Vous pouvez trouver la documentation des queries fuzzy ici et wildcard ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "wildcard" : {
      "text" : "ameri*a"
    }
  }
}'
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "fuzzy" : {
      "text" : "ameriqa"
    }
  }
}'

8. Chercher des documents (3/3)

Faire une recherche sur l'ensemble de l'index en faisant une facet qui donne le count des 20 termes les plus fréquents dans le champ text

Hint
Vous pouvez trouver la documentation de la facet terms ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "match_all" : { }
  }, 
  "facets" : {
    "countTerms" : {
      "terms" : {
        "field" : "text",
        "size": 20
      }
    }
  }
}'

9. Altérer la réponse

Faire la recherche pour trouver le document qui contient america tout en formattant la réponse pour qu'elle soit lisible humainement, et afficher l'explication du scoring.

Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?pretty=true' -d '{
  "explain": true,
  "query" : {
    "match" : {
      "text" : "america"
    }
  }
}'

12. Installer le plugin marvel et HEAD

Le plugin marvel vous permet de surveiller ce qu'il se passe sur le cluster et tour mieux comprendre de clustering le plugin HEAD permet de visualiser la répartition des index sur les différentes instances. (N'oubliez pas de redémarrer après avoir installer les plugins !)
  • marvel : Homepage
  • head : Homepage
Solution 
$ cd elasticsearch-1.1.0/bin
$ ./plugin -install elasticsearch/marvel/latest
$ ./plugin -install mobz/elasticsearch-head
Un fois installé, vous pouvez accéder à l'interface du plugin head ici : http://localhost:9200/_plugin/head/ et pour marvel ici : http://localhost:9200/_plugin/marvel/

Relancer elasticsearch plusieurs fois et aller voir sur head et marvel les shards se répartisent sur les différentes instances.

14. Afficher le mapping

Afficher le mapping de l'index wikipedia pour s'assurer du type de chacun des champs.

Solution 
$ curl -XGET 'http://23.251.133.134:9200/wikipedia/_mapping'

15. Chercher des documents (avancé 1/2)

Ecrire une requête qui retourne les articles dont leur titre contient america et qui ne sont pas des redirections.

Hint
Il faut faire un filter pour filtrer les articles qui ne sont pas des redirections.
Solution 
$ curl -XGET 'http://23.251.133.134:9200/wikipedia/_search?pretty=true' -d '{
  "query" : {
    "match" : {
      "title" : "america"
    }
  },
  "filter" : {
        "term" : {
         "redirect" : 0
       }
  }
}'
